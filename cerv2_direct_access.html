CERV2 Module Direct Access Test

This page helps test direct access to the CERV2 module without redirection to the client portal.

We've fixed the issue where all module links were being redirected to /client-portal, making it impossible to directly access the CERV2 module by URL. This was caused by a global navigation rewrite in the client's router configuration that was too broad.

Technical Summary of Fixes

The following changes were made to resolve this issue:

// Previous navigation code in ClientPortalLanding.jsx:
onClick={() => {
  console.log('Navigating to:', module.path);
  // Use complete URL with origin
  const fullUrl = window.location.origin + module.path;
  console.log('Full URL:', fullUrl);
  window.location.href = fullUrl;
}}

// Fixed navigation code:
onClick={() => {
  console.log('Navigating to:', module.path);
  // Navigate directly to the module path without forcing client-portal prefix
  window.location.href = module.path;
}}

We also removed the misleading console log message that was incorrectly stating: "All module access links updated to point to /client-portal".
