Authentication

All API requests require authentication using an API key. Include this key in the header of each request.

API Key Authentication

Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY
Example Request
curl -X GET \
  "https://api.lumentrial.ai/v1/csr/search?query=COPD" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

CSR Endpoints

GET /api/csr/search

Search for CSR reports based on various criteria.

Query Parameters
  • query - Text search across all fields
  • therapeutic_area - Filter by therapeutic area
  • phase - Filter by study phase
  • sponsor - Filter by sponsor name
  • from_date - Start date in YYYY-MM-DD format
  • to_date - End date in YYYY-MM-DD format
  • limit - Maximum number of results (default: 20, max: 100)
  • offset - Pagination offset (default: 0)
Example Request
curl -X GET \
  "https://api.lumentrial.ai/v1/csr/search?query=COPD&phase=Phase+3&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
GET /api/csr/{id}

Get detailed information about a specific CSR report.

Path Parameters
  • id - Unique identifier of the CSR report
Example Request
curl -X GET \
  "https://api.lumentrial.ai/v1/csr/12345" \
  -H "Authorization: Bearer YOUR_API_KEY"
GET /api/csr/{id}/sections

Get structured sections of a specific CSR report.

Path Parameters
  • id - Unique identifier of the CSR report
Query Parameters
  • section - Optional filter for specific section (e.g., "study_design", "results", "safety")
Example Request
curl -X GET \
  "https://api.lumentrial.ai/v1/csr/12345/sections?section=results" \
  -H "Authorization: Bearer YOUR_API_KEY"

Webhooks

Receive notifications when new CSR reports are added or analysis is complete.

POST /api/webhooks

Create a new webhook subscription.

Request Body
{
  "url": "https://your-server.com/webhook-receiver",
  "events": ["csr.created", "csr.updated", "analysis.complete"],
  "secret": "your_signing_secret"
}
Example Request
curl -X POST \
  "https://api.lumentrial.ai/v1/webhooks" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.com/webhook-receiver",
    "events": ["csr.created", "csr.updated", "analysis.complete"],
    "secret": "your_signing_secret"
  }'
GET /api/webhooks

List all webhook subscriptions.

Example Request
curl -X GET \
  "https://api.lumentrial.ai/v1/webhooks" \
  -H "Authorization: Bearer YOUR_API_KEY"
DELETE /api/webhooks/{id}

Delete a webhook subscription.

Path Parameters
  • id - Unique identifier of the webhook subscription
Example Request
curl -X DELETE \
  "https://api.lumentrial.ai/v1/webhooks/12345" \
  -H "Authorization: Bearer YOUR_API_KEY"

Error Handling

The API uses standard HTTP response codes to indicate success or failure.

HTTP Status Codes

  • 200 OK - The request was successful
  • 201 Created - A new resource was successfully created
  • 400 Bad Request - The request was malformed or had invalid parameters
  • 401 Unauthorized - Authentication failed or token was missing
  • 403 Forbidden - The authenticated user doesn't have permission for the requested operation
  • 404 Not Found - The requested resource was not found
  • 429 Too Many Requests - Rate limit exceeded
  • 500 Internal Server Error - An error occurred on the server

Error Response Format

All error responses use the following JSON format:

{
  "error": {
    "code": "invalid_parameter",
    "message": "The parameter 'therapeutic_area' is invalid",
    "details": {
      "parameter": "therapeutic_area",
      "valid_values": ["Oncology", "Cardiology", "Neurology", ...]
    }
  }
}

Rate Limits

API requests are subject to rate limiting to ensure fair usage and stability.

Plan Rate Limit Burst Limit
Free 100 requests per day 10 requests per minute
Basic 1,000 requests per day 60 requests per minute
Premium 10,000 requests per day 300 requests per minute
Enterprise Custom Custom

Rate Limit Headers

API responses include the following rate limit headers:

  • X-RateLimit-Limit - The maximum number of requests allowed in the current time window
  • X-RateLimit-Remaining - The number of requests remaining in the current time window
  • X-RateLimit-Reset - The time at which the current rate limit window resets, in UTC epoch seconds
